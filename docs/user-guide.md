# AuctorForge 使用说明

AuctorForge 是面向中文网文作者的本地优先 AI 长篇创作工作台。它不是“把整本书交给 AI 代写”的工具，而是帮助你把项目资料、设定、大纲、正文、审查意见、助手对话和模型配置放在同一个工作流里。

这份说明面向第一次试用的作者。建议先用示例项目熟悉流程，再决定是否导入真实稿件。

## 1. 启动应用

先按 [快速开始](quick-start.md) 启动本地 API 服务和 Web UI：

```bash
pnpm dev:server
pnpm dev:web
```

打开 Vite 输出的地址，通常是：

```text
http://localhost:5173
```

进入首页后，你会看到新建、导入、试用示例项目、最近项目和模型配置入口。

![AuctorForge 首页](assets/screenshots/user-guide-home.png)

## 2. 第一次试用

第一次打开时，建议点击 **试用示例项目**。

示例项目使用虚构内容，不需要你放入真实稿件。这样可以先确认：

- 项目文件保存在哪里
- 编辑器和助手对话如何配合
- 模型配置入口在哪里
- 哪些功能只是本地编辑，哪些功能可能调用远程模型

如果你已经有项目，也可以在首页的 **最近项目** 中选择并继续。

## 3. 理解工作台

打开项目后，AuctorForge 会进入创作工作台。一个典型工作区包含：

- 顶部栏：返回首页、查看当前项目、进入模型配置。
- 文稿区域：打开和编辑本地项目文件，例如 `PROJECT.md`、设定、大纲、章节草稿。
- 创作助手：围绕当前文档和项目上下文讨论想法、推进流程或生成建议。
- 状态提示：显示当前文件、保存状态、流程阶段和模型配置状态。

![AuctorForge 工作台](assets/screenshots/user-guide-workspace.png)

文稿区域里的内容是可编辑的草稿材料。你可以像编辑普通 Markdown 文档一样修改它。助手可以给建议，但最终写入、修改和定稿仍由作者决定。

## 4. 推荐使用流程

如果你是第一次用 AuctorForge，可以按这个顺序试：

1. 打开示例项目。
2. 阅读 `PROJECT.md`，理解项目摘要、角色、设定、大纲和当前状态。
3. 在右侧创作助手中输入一个轻量请求，例如：

```text
帮我总结当前项目的主角目标和下一步剧情风险。
```

4. 查看助手回答是否符合你的写作习惯。
5. 尝试修改一小段示例文本。
6. 确认保存状态。
7. 再决定是否新建自己的项目或导入旧稿。

不要一开始就导入完整真实稿件。先用一小段非敏感文本或虚构内容试跑，更容易判断工具是否适合你。

## 5. 模型配置

首页和工作台顶部都有 **模型配置** 入口。

在没有配置模型服务前，你仍然可以：

- 浏览项目
- 编辑本地文档
- 查看示例项目
- 理解工作台结构

只有当你使用需要模型能力的功能时，相关文本才可能被发送给你配置的模型服务商。配置前请先阅读 [稿件隐私与安全](privacy.md)。

## 6. 稿件安全

网文作者最重要的资产是未公开作品。使用真实稿件前，请先确认这些边界：

- 项目资料应保存在你知道位置的本地文件夹。
- 大改前复制整个项目文件夹做备份。
- 截图、Issue、讨论区反馈只使用虚构或脱敏内容。
- 不要公开 API key、模型凭证、未发布正文或个人本地路径。
- 调用远程模型前，先理解会发送哪些文本。

AuctorForge 的方向是本地优先和配置透明，但这不等于所有 AI 调用都只在本机发生。你使用哪个模型服务，文本就可能发送给对应服务。

## 7. 适合反馈的问题

如果你想给项目提建议，最有帮助的是具体写作场景：

- 你平时怎么管理人物、设定、大纲和章节？
- 哪个环节最容易前后矛盾？
- 你希望 AI 给审查意见、续写建议，还是只做整理？
- 你最担心稿件泄露、版权、风格跑偏，还是模型不好控制？
- 你希望导出成 Markdown、TXT，还是其他格式？

可以通过 GitHub Issue 反馈：

- [作者工作流反馈](https://github.com/Te9ui1a/AuctorForge/issues/4)
- [中文长篇写作模板贡献](https://github.com/Te9ui1a/AuctorForge/issues/3)
- [截图和演示素材改进](https://github.com/Te9ui1a/AuctorForge/issues/5)

请只使用虚构或脱敏示例，不要贴真实未公开稿件。

## 8. 常见问题

### AuctorForge 会不会替我写完整本书？

项目目标不是替作者接管创作，而是帮助作者管理材料、获得修改建议、复盘章节、整理上下文和减少长篇一致性问题。

### 没有配置模型，可以用吗？

可以。你仍然可以查看示例项目、浏览本地文件和编辑文档。需要 AI 能力的功能才依赖模型配置。

### 可以直接导入真实稿件吗？

技术上可以逐步支持，但第一次试用不建议这样做。先使用示例项目或非敏感文本确认流程、保存位置和模型调用边界。

### 截图或反馈可以贴自己的正文吗？

不建议。公开反馈请使用虚构、改写或脱敏内容，避免泄露未发布稿件。
