# 稿件隐私与安全 / Privacy And Manuscript Safety

AuctorForge 面向作者，所以未公开稿件和项目资料需要被认真对待。

## 哪些内容可能敏感

- 正文章节
- 大纲
- 人物卡
- 世界观与设定
- 审稿意见
- 包含故事细节的提示词模板
- 模型 API key
- 本地项目路径
- 请求日志

## 本地文件

AuctorForge 使用本地项目文件和工作流资产。使用有价值的真实稿件前，请先确认：

- 项目文件夹在哪里
- 应用会创建或编辑哪些文件
- 是否已有备份
- 如何导出或迁移你的作品

第一次打开时，AuctorForge 可以创建虚构示例项目 `Lantern Road`。建议先用这个示例检查工作台、文件树和编辑器，再导入真实稿件。

## 远程模型服务商

如果某个功能调用远程模型服务，服务商可能会收到项目中的部分文本。具体可能包括：章节草稿、大纲、人物笔记、审稿说明或聊天消息。

启用模型服务商前，请先：

- 阅读服务商的数据政策
- 用虚构内容测试
- 在信任配置前，不要粘贴完整私人稿件
- 不要把 API key 提交到 Git

即使未配置模型服务商，你仍然可以浏览和编辑本地项目。

## 公开仓库检查清单

发布或分享仓库前，请检查：

- `.env`
- API keys 和 tokens
- 未公开稿件材料
- 本地项目文件夹
- 包含请求内容的测试日志
- 包含私人文本的截图
- 个人本地路径

## 维护者备注

未来如果加入托管或同步功能，必须说明：

- 上传了什么
- 由谁处理
- 保留多久
- 是否用于训练
- 用户如何删除
