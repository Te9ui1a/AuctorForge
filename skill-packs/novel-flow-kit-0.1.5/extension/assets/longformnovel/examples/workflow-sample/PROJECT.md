# PROJECT.md - Workflow Sample

> 这是 AuctorForge 内置示例项目。它像真实项目一样存放在项目目录里，用于演示从创意、设定、大纲、正文、审查到记忆文件的协作方式。

## 1. 项目基因
- 类型/题材：都市异能
- 核心主题：普通人在压力中学会正确使用能力，而不是被能力拖着走。
- 目标读者：喜欢现实都市、轻悬疑、能力成长和团队羁绊的网文读者。
- 情感承诺：压迫感、爽点、反转、逐章推进。
- 字数目标 / 里程碑计划：示例只展示 1 卷 3 章结构。

## 2. 世界体系
- 城市：雾港市
- 规则摘要：少数人会在强烈情绪或高压环境中觉醒“回声”能力，能力会留下可追踪痕迹。
- 设定文件：`2-设定/2.2_新书设定案.md`
- 金手指文件：`2-设定/2.3_金手指设定.md`

## 3. 角色
- 林澈：外卖骑手，能听见物品残留的“最后一句话”。
- 周棠：急诊科医生，知道觉醒事件背后的第一批线索。
- 许闻舟：城市安全顾问，表面帮忙，暗中试探林澈。
- 角色总表：`2-设定/2.4_主要角色设定表.md`

## 4. 情节与结构
- 总纲：`3-大纲/3.1_全书结构总纲.md`
- 第01卷卷纲：`3-大纲/第01卷_完整卷纲.md`
- 第01卷章纲：`3-大纲/第01卷_章纲.md`

## 5. 风格指南
- 叙事口吻：克制、贴近生活、动作和细节优先。
- 去 AI 味要求：少总结、少口号、少解释性旁白，多用场景让读者自己判断。

## 6. 重要备注与软性约束
- 不写无成本开挂；能力使用必须付出代价或暴露风险。
- 不让反派降智；每次冲突都要符合其信息范围内的最优选择。
- 不用审查拦截正文；审查只给修改意见，最终通过定稿流程落盘。
