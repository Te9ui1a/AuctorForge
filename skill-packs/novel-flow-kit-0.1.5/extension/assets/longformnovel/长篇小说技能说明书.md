> 📚 一套基于工业化网文创作流程的 NovelFlow Skills  
> 我们的理念：创意和思路没有定式，但是网文创作流程，是有那么几套符合故事创作的最优解的。这是其中之一。

## 🔧 前置要求

### 建议
- ✅ 使用Cursor、NovelFlow 或者类似IDE 软件，能够读取和写入文件。

## 🚀 快速开始

### 第一步：初始化项目

技能初始化之后，会创建项目总控文件 `PROJECT.md`
同时，会自动创建以下目录结构：
```
.novelkit/          # 记忆库（角色状态、伏笔追踪等）、宪法
1-边界/             # 样板书与分析结果
2-设定/             # 设定案
3-大纲/             # 总纲、卷纲与章纲
4-正文/             # 正文草稿
5-审查/             # 质检报告
```

### 第二步：选择起点
根据交互提示，可以选择以下三种起点：
1、**标准模式**
就是从零开始定义方向，生成文风、套路、节奏等，然后进行创意脑暴，生成设定、角色、金手指等，然后进行大纲规划，最后进行正文写作。
2、**灵活模式**
就是导入旧稿或从特定点切入，然后进行后面的创作。
3、**参考模式**
就是根据样板书分析，生成文风、套路、节奏等，然后进行后面的创作。

选择后，技能包会自动执行相应的子技能，并提示下一步操作。

### 第三步：按提示推进
每个技能完成后会自动提示下一步，或随时输入：
- `检查进度` - 查看当前状态
- `下一步做什么` - 获取推荐操作

## ⚠️ 注意

- Token 消耗警告：拆解你自己的样板书的时候，章节数过多可能消耗数十万甚至数百万Token
（长篇小说曾试过，拆解200 章左右，花费几美金。当然，跟不同的模型有关。）

## 💡 使用技巧

**灵活使用**：
- ✅ **PROJECT 是核心**：所有状态都记录在 PROJECT.md，定期查看 §7 和 §8
- ✅ **记忆文件很重要**：`.novelkit/memory/` 下的文件记录状态和伏笔，不要手动删除
- ✅ **渐进式**：写完第一卷章纲 → 写第一卷正文 → 再写第二卷卷纲（推荐）
- ✅ **批量式**：先写完所有卷的大纲 → 再开始正文创作
- ⚠️ 不建议使用快速启动式：只写前几章章纲 → 立即开始写作 → 边写边补充
- ✅ **大纲可分批**：不必一次性写完所有卷的大纲，可以边写边规划
- ✅ **Review 随时插入**：写完大纲可以质检，写完正文也可以质检

- 如果你不小心删除了模板文件，又发现无法初始化，可以采用以下命令，帮你重新写入模板文件：

1. 打开命令面板
macOS：Cmd + Shift + P
Windows / Linux：Ctrl + Shift + P
2. 在列表里找这条命令
在命令面板顶部的输入框里输入：「长篇小说」 或 「初始化」，列表中会出现：
长篇小说: 初始化长篇小说流程 Kit
点这一项就会执行初始化（复制技能包、创建目录、复制 PROJECT.md 等）。

## 📞 获取帮助

如果初始化、打开项目或推进流程时遇到问题，建议先按下面顺序排查：

1. 先查看 AuctorForge 的 `README.md` 和 `docs/quick-start.md`，确认本地服务、Web UI、项目目录和模型配置都已按当前版本启动。
2. 如果问题和稿件隐私、模型调用或数据保存位置有关，先阅读 `docs/privacy.md`，确认哪些内容只在本地处理，哪些功能可能发送给模型服务。
3. 如果是技能包流程卡住，可以检查 `PROJECT.md`、`.novelkit/`、`1-边界/` 到 `5-审查/` 这些目录是否还在，并查看 `PROJECT.md` 中记录的当前阶段、下一步和待处理项。
4. 如果确认是项目问题，请通过仓库 Issue 反馈。反馈时尽量写清楚：你想完成的步骤、预期结果、实际结果、复现步骤、系统和浏览器版本、Node.js / pnpm 版本，以及是否配置了模型服务。
5. 不要在公开反馈里粘贴未发布正文、完整设定、商业稿件或真实 API Key。需要举例时，请使用虚构或脱敏内容。
