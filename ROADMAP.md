# Roadmap

This roadmap keeps open-source readiness and writer value moving together.

## Phase 1: Open-Source Ready

Goal: a new developer or writer can understand the project and run it locally.

- [x] Add README, license, contribution guide, security notes, and issue templates
- [x] Add `.env.example`
- [x] Add screenshots or a short demo GIF
- [ ] Confirm all public docs avoid private project names, personal names, and old working titles
- [ ] Add a first-release checklist
- [x] Add a short architecture overview
- [ ] Decide whether to initialize or connect the top-level Git repository before publishing

## Phase 2: Writer Trial Flow

Goal: a Chinese long-form fiction writer can complete a small safe trial without using private manuscript material.

- [ ] Add a fictional sample project
- [ ] Document the first-run flow with screenshots
- [ ] Explain where local project files are created
- [ ] Explain model-provider setup and request boundaries
- [ ] Add a guided "create/open project" walkthrough
- [ ] Add export and backup guidance

## Phase 3: Trust And Control

Goal: writers can see what the app will change or send before it happens.

- [ ] Show model request scope clearly in the UI
- [ ] Add stronger project backup and export affordances
- [ ] Improve local settings documentation
- [ ] Add privacy-centered release notes
- [ ] Review generated project files for public-safe defaults

## Phase 4: Workflow Depth

Goal: make AuctorForge more useful for real long-form writing.

- [ ] Expand reusable workflow templates
- [ ] Improve continuity and consistency checks
- [ ] Improve chapter review loops
- [ ] Add better project memory summaries
- [ ] Support template import/export

## OpenSpec Direction

Use OpenSpec for product behavior changes. Suggested future changes:

- `add-author-onboarding`
- `add-sample-project`
- `add-model-request-transparency`
- `add-export-and-backup-controls`
- `add-template-library-management`
